# =============================================================================
# REQUIREMENTS.TXT - Python Dependency Management for Demand Forecasting System
# =============================================================================
#
# PURPOSE:
# This file centralizes all Python package requirements for the demand forecasting
# web application. It ensures consistent environments across development, testing,
# and production deployments.
#
# USAGE LOCATIONS:
# 1. Local Development:
#    - setup-local.bat: Installs dependencies into virtual environment
#    - README.md: Manual setup instructions for developers
#    - Ensures all team members use identical package versions
#
# 2. Docker Container Build:
#    - Dockerfile: Installs dependencies during container image creation
#    - Uses --no-cache-dir for smaller container images
#    - Supports both root-level and subdirectory deployment
#
# 3. Cloud Deployment (Render):
#    - render.yaml: Automated build process installs dependencies
#    - Ensures production environment matches development exactly
#    - Enables one-click deployment to Render's cloud platform
#
# 4. CI/CD and Testing:
#    - Automated pipelines install from this file
#    - Guarantees consistent testing environments
#    - Prevents "works on my machine" issues
#
# VERSION PINNING STRATEGY:
# Uses exact versions (==) rather than ranges for:
# - Predictability: Eliminates surprises from automatic updates
# - Stability: Prevents breaking changes in production
# - Debugging: Easier to reproduce issues with known versions
# - Security: Deliberate updates rather than automatic patches
#
# DEPENDENCY CATEGORIES:
# =============================================================================
